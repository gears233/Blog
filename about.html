这是一条闲鱼的博客网站

本来当初建站的时候就想着要少折腾，不过最后确实又折腾了不少东西 = =

说实话，博客托管在 Google Blogger 上，我仍然觉得是个不错的选择，尤其是对我这种编程小白来说，但是由于各种原因，国内访问总是可能会遇到一点障碍，另外随着我折腾的东西变多，发现了一个难以避免的问题——网站整体变得愈发的累赘了。

出于对网站页面管理的自由的追求与自身怠惰的现状，于是，在我面前的选择其实只有两种。继续用 Blogger 或者 转投到 GitHub Pages 的怀抱。突然，一句话回响在我脑海之中——小孩子才做选择，我全都要！

简单地看了下，最终尝试选用Hugo来做博客了，刚上手没几天，仍然还在摸索中。静态网站的话，目前大家用的比较多的应该还是 Hexo 吧，中文方面的支持文档比较多。不过，最后我还是选用了 Hugo，主要是看中了 ta 的简洁，单二进制文件使我不需要去额外安装一堆东西，另外就是渲染页面速度确实也很快。不过由于我选的主题的原因，生成页面的时候需要用 HUGO_ENV=production hugo –gc –minify 命令，怎么也记不住，干脆就放在这里，老老实实复制粘贴吧。

今天，终于把原先博客上的内容搬迁转移地差不多了。图床，我一直觉得是个深坑，既然用了 GitHub Pages 了，那就图床也用 ta 吧（有种网站和图片共存亡的感觉2333）.搬迁的过程，由于有个转换成 Markdown 的过程，文章某些局部还是需要手动修改的，确实累…先立个 Flag，没特殊情况，以后多半不会再搬迁了。不过，还有个问题一直没有解决，网站的域名问题。诚如你所看到的，目前用的其实是个二级域，在搜索引擎收录方面是很吃亏的，尤其国内的那些几乎都是不予收录。一方面是真的还没有想好买个什么样的顶级域比较合适，me 或者 xyz 之类的看上去应该是不错的选择；另一方面更多的是也没有想好网站是否要去备案，如果备案又可能有各种各样的风险。还有一部分的担心是域名目前似乎最多也只能买个10年的时间，以后要是倦了没想着再续费啥的说不定这些内容就再也不能被人看到了，所以暂且搁置着，等到合适的时机，我会做出决断的…（似乎又立了个 Flag）

之前网站的内容还是托管在 Google Blogger 上，正常保留，只是以后不会在原先网站上更新了。姑且将原先的网站称为 Refun 1.0 吧，DNS 解析稍微改了下设置，原先的网站现在请访问 https://blog.refun.eu.org
